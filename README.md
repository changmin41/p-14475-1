# 덧셈과 뺄셈을 위한 고민과정
 
### 인덱스 2칸씩 점프하며 계산
 
배열 구조: [숫자, 연산자, 숫자, 연산자, 숫자, ...]
 
```
배열: [3, +, 5, -, 2]
      0  1  2  3  4
 
result = 3 (첫 번째 숫자)
 
for (int i = 1; i < length; i += 2) {
    i=1: operator = "+", num = 5 → result = 8
    i=3: operator = "-", num = 2 → result = 6
}
```
 
### 이 방식이 가능한 이유
 
곱셈 처리 후 배열이 정리됨
 
```
원본: [10, +, 5, *, 2, +, 3]
곱셈 처리: [10, +, 10, +, 3]  ← 5*2를 계산해서 빈 공간 제거
 
결과: [숫자, 연산자, 숫자, 연산자, ...] 형태
      i += 2로 연산자와 숫자가 정확히 매칭됨
```

# 괄호 처리를 위한 고민과정

### 핵심 고민

괄호를 어떻게 처리할 것인가

방식 선택
- 방식 A: 완전 재귀 (removeParentheses() 별도 함수)
  순수하지만 단항 연산자, 공백 처리가 복잡
- 방식 B: while 루프 + 부분 재귀 (run() 호출)  
  간단하고 기존 로직 100% 재사용

선택: 방식 B

### 괄호 처리 로직

```
"10 + (20 - (3 + 4))"에서
→ 가장 안쪽 (3 + 4)부터 찾기
→ run(inner)로 재귀 계산
→ 결과로 괄호 교체
→ 반복
```

### 최종 구조

```
while (expression.contains("(")) {
    1. 가장 안쪽 () 찾기
    2. run(inner)로 재귀 계산
    3. 괄호를 결과로 교체
}
// 기존 계산 로직 (곱셈 → 덧셈/뺄셈)
```

### 선택 이유

기존 코드 재사용으로 t1~t17 로직 유지
run() 내부에서 재귀 호출로 조건 만족
while 루프만 추가해서 간단함
